# 工部: Spatial Simulation and Model Engineering

Use with `兴工：实验推演` for PLUS, InVEST, Circuitscape, Linkage Mapper, OSMnx, landscape metrics, and scenario simulation.

## Required Outputs

1. Model purpose: prediction, mechanism explanation, optimization, or scenario comparison.
2. Inputs: each layer, year, resolution, preprocessing, source.
3. Model flow: ordered stages from raw data to result.
4. Parameter table: threshold, weight, hyperparameter, source/rationale.
5. Validation: accuracy, sensitivity, robustness, comparison model.
6. Interpretation: what the model can and cannot claim.

## Examples of Good Specificity

For PLUS:

`LUCC 2010/2020 -> driving factors -> LEAS rule mining -> CARS simulation -> accuracy validation -> scenario prediction -> ecological network reconstruction`.

For Circuitscape:

`source identification -> resistance surface -> current density -> pinch points -> barrier detection -> sensitivity test of resistance weights`.
