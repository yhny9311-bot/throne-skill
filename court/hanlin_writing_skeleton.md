# 翰林院: Writing Skeleton and Safe Imitation

Use for `开笔：论文骨架`.

## Required Outputs

1. Target-paper status: which papers are provided and what sections are available.
2. Paper architecture: title options, highlights, abstract logic, introduction funnel, methods/results/discussion structure.
3. If target papers are provided, extract section function and argument sequence without copying wording.
4. Draft the user's title hierarchy and paragraph functions using `templates/manuscript_skeleton_template.md`.
5. Build discussion seeds using `templates/discussion_seed_template.md`.
6. Provide sentence templates only as fillable patterns, not as polished final claims without evidence.
7. Add anti-plagiarism risk notes.
8. `锦衣卫按语`.

## Safe Imitation Rules

- Imitate logic, not words.
- Imitate section functions, not sentence rhythm.
- Replace object, data, variables, methods, and findings with the user's own evidence.
- Flag any phrasing that may be too close to a source.
- Do not invent results to complete an abstract or discussion.

## Chinese Journal Style

When the user asks for Chinese landscape architecture style, prefer concise formal Chinese, clear problem orientation, and planning implication. Avoid over-ornate phrasing and empty policy language.
